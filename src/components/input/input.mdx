# Input

The input component is by design just a styled html / react component to allow developers maximum flexibility to use whatever form library they choose, or simply by listening on the native `onChange` event, accessing the `value` property of the input.

> For all users, we recommend combining with the `InputGroup` and `FormLayout` components to ensure a consistent responsive layout, spacing and combining with labels, error messages and icons. See separate stories for more information.

You may also wish to add your own custom elements eg a JS Datepicker - in this case, just target the default input with your own custom component or use the CSS classes provided.
